You must substitute the {\ttfamily \texorpdfstring{$<$}{<}values\texorpdfstring{$>$}{>}} with your own settings.

Parameters are split into two halves separated by a colon. The left side represents the host and the right side the container.

{\bfseries{Example\+:}} {\ttfamily -\/p external\+:internal} -\/ This shows the port mapping from internal to external of the container. Therefore {\ttfamily -\/p 47990\+:47990} would expose port {\ttfamily 47990} from inside the container to be accessible from the host\textquotesingle{}s IP on port {\ttfamily 47990} (e.\+g. {\ttfamily \href{http://}{\texttt{http\+://}}\texorpdfstring{$<$}{<}host\+\_\+ip\texorpdfstring{$>$}{>}\+:47990}). The internal port must be {\ttfamily 47990}, but the external port may be changed (e.\+g. {\ttfamily -\/p 8080\+:47990}). All the ports listed in the {\ttfamily docker run} and {\ttfamily docker-\/compose} examples are required.

When using data volumes (-\/v flags) permissions issues can arise between the host OS and the container. To avoid this issue you can specify the user PUID and group PGID. Ensure the data volume directory on the host is owned by the same user you specify.

In this instance {\ttfamily PUID=1001} and {\ttfamily PGID=1001}. To find yours use id user as below\+:

\DoxyCodeLine{uid=1001(dockeruser)\ gid=1001(dockergroup)\ groups=1001(dockergroup)}

\end{DoxyCode}
