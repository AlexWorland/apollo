The recommended method for running Sunshine is to use the binaries included in the \href{https://github.com/LizardByte/Sunshine/releases/latest}{\texttt{latest release}}, unless otherwise specified.

\href{https://github.com/LizardByte/Sunshine/releases}{\texttt{Pre-\/releases}} are also available. These should be considered beta, and release artifacts may be missing when merging changes on a faster cadence.\hypertarget{md_getting__started_autotoc_md20}{}\doxysection{\texorpdfstring{Binaries}{Binaries}}\label{md_getting__started_autotoc_md20}
Binaries of Sunshine are created for each release. They are available for Linux, mac\+OS, and Windows. Binaries can be found in the \href{https://github.com/LizardByte/Sunshine/releases/latest}{\texttt{latest release}}.

\begin{DoxyNote}{Note}
Some third party packages also exist. See \doxysectlink{md_third__party__packages}{Third Party Packages}{0} for more information. No support will be provided for third party packages!
\end{DoxyNote}

\begin{DoxyItemize}
\item On Windows, Sunshine uses the Desktop Duplication API which only supports capturing from the GPU used for display. If you want to capture and encode on the e\+GPU, connect a display or HDMI dummy display dongle to it and run the games on that display.
\item When an application is started, if there is an application already running, it will be terminated.
\item If any of the prep-\/commands fail, starting the application is aborted.
\item When the application has been shutdown, the stream shuts down as well.
\begin{DoxyItemize}
\item For example, if you attempt to run {\ttfamily steam} as a {\ttfamily cmd} instead of {\ttfamily detached} the stream will immediately fail. This is due to the method in which the steam process is executed. Other applications may behave similarly.
\item This does not apply to {\ttfamily detached} applications.
\end{DoxyItemize}
\item The "{}\+Desktop"{} app works the same as any other application except it has no commands. It does not start an application, instead it simply starts a stream. \item For the Linux flatpak you must prepend commands with {\ttfamily flatpak-\/spawn -\/-\/host}.
\item If inputs (mouse, keyboard, gamepads...) aren\textquotesingle{}t working after connecting, add the user running sunshine to the {\ttfamily input} group.

Streaming HDR content is officially supported on Windows hosts and experimentally supported for Linux hosts.


\begin{DoxyItemize}
\item General HDR support information and requirements\+:
\begin{DoxyItemize}
\item HDR must be activated in the host OS, which may require an HDR-\/capable display or EDID emulator dongle connected to your host PC.
\item You must also enable the HDR option in your Moonlight client settings, otherwise the stream will be SDR (and probably overexposed if your host is HDR).
\item A good HDR experience relies on proper HDR display calibration both in the OS and in game. HDR calibration can differ significantly between client and host displays.
\item You may also need to tune the brightness slider or HDR calibration options in game to the different HDR brightness capabilities of your client\textquotesingle{}s display.
\item Some GPUs video encoders can produce lower image quality or encoding performance when streaming in HDR compared to SDR.
\end{DoxyItemize}
\end{DoxyItemize}

Additional information\+:

@tabs\{ @tab\{ Windows \texorpdfstring{$\vert$}{|}
\begin{DoxyItemize}
\item HDR streaming is supported for Intel, AMD, and NVIDIA GPUs that support encoding HEVC Main 10 or AV1 10-\/bit profiles.
\item We recommend calibrating the display by streaming the Windows HDR Calibration app to your client device and saving an HDR calibration profile to use while streaming.
\item Older games that use NVIDIA-\/specific NVAPI HDR rather than native Windows HDR support may not display properly in HDR. \}
\end{DoxyItemize}

@tab\{ Linux \texorpdfstring{$\vert$}{|}
\begin{DoxyItemize}
\item HDR streaming is supported for Intel and AMD GPUs that support encoding HEVC Main 10 or AV1 10-\/bit profiles using VAAPI.
\item The KMS capture backend is required for HDR capture. Other capture methods, like Nv\+FBC or X11, do not support HDR.
\item You will need a desktop environment with a compositor that supports HDR rendering, such as Gamescope or KDE Plasma 6.
\end{DoxyItemize}
